txm-A Tmux Helper Tool

txm is a command-line utility designed to enhance your experience with tmux, a popular terminal multiplexer. with txm, you can streamline common tmux tasks, manage sessions and panes efficiently, and execute commands seamlessly.

Installation

To install txm, you can either build from the source or install it from pypi repository

PyPi

pip install txm

Build from source

You can do the following steps:

  1. Clone the txm repository from GitHub.

  2. Navigate to the project directory:

    cd txm

  3. Install txm in editable mode using pip:

pip install -e .

This will make the txm command available in your terminal.

Getting Started

Session Management

Create a New Session

Use txm new to create a new tmux session. Provide a session name as an argument. If no name is provided, a default name will be used.

Example:

txm new mysession

List All Sessions

To view a list of all available tmux sessions, use the txm list command.

Example:

txm list

Attach to a Session

To attach to an existing tmux session, use txm attach followed by the session name.

Example:

txm attach mysession

Detach from a Session

To detach from the current tmux session, use the txm detach command.

Example:

txm detach

Pane Management

Split Pane Vertically

To split the current tmux pane vertically, use the txm vsplit command.

Example:

txm vsplit

Split Pane Horizontally

To split the current tmux pane horizontally, use the txm hsplit command.

Example:

txm hsplit

Navigate Between Panes

Navigate between tmux panes using the txm navigate command, followed by a direction (U for up, D for down, L for a left, R for right).

Example:

txm navigate U

This will navigate to the pane above.

Command Execution

Execute a Command

Use txm run followed by a command to execute it in the current tmux pane.

Example:

txm run "ls -l"

Configuration

txm offers customization options through configuration files.

txm-configjson

The .txm-config.json file allows you to specify various settings for txm. It should be placed in the root directory of your project or your home directory. Here is a default template:

{ "tmux_path": "/usr/local/bin/tmux", "log_directory": "logs/", "default_session_name": "my_session" }

  • tmux_path: Specifies the path to the tmux executable.

  • log_directory: Specify the directory where tmux pane logs will be saved.

  • default_session_name: Specifies the default name for a new tmux session.

txm-presetsjson

The .txm-presets.json file stores saved tmux session presets. You can create it manually or use the txm save command to generate it. Here is a default template:

{ "presets": [ { "name": "preset_1", "windows": [ { "name": "window_1", "panes": ["command1", "command2"] }, { "name": "window_2", "panes": ["command3", "command4"] } ] } ] }

With

  • name: Specifies the name of the saved preset.

  • windows: An array of windows in the preset.

    • name: Name of the window.

    • panes: An array of commands to run in each pane.

Aliases

Aliases allow you to define shorthand commands for txm actions. You can create and manage aliases in the .txm-config.json file.

Example:

{ "aliases": { "c": "new", "a": "attach", "l": "list", "d": "detach" } }

With these aliases, you can use txm c, txm a, txm l, and txm d as shortcuts for txm new, txm attach, txm list, and txm detach, respectively.

Presets

Presets allow you to define and save complex tmux session configurations. You can create and manage presets using the txm preset command.

Example:

# Create a preset named "work" with specific windows and panes txm preset create work --window "editor" "vim" "top" --window "logs" "tail -f logs.txt" # Load the "work" preset to set up the tmux session txm preset load work

Logging

You can enable logging for tmux panes to capture terminal output. Logs are saved in the specified log directory (configured in .txm-config.json).

Example:

# Start logging for the current pane txm log start # Stop logging for the current pane txm log stop # Show logs for the current pane txm log show

Pane Synchronization

Pane synchronization allows you to synchronize input across multiple tmux panes. Use the txm sync command to enable or disable synchronization.

Example:

# Enable pane synchronization txm sync enable # Disable pane synchronization txm sync disable
